# 一部依存関係のバージョン固定（理由不明）

## 概要

ほとんどの依存関係はキャレット（`^`）でバージョン範囲を指定しているが、一部のパッケージだけが厳密な固定バージョンになっており、その理由がコード上に記載されていない。

## 現状

`package.json` で固定バージョン指定されているパッケージ：

| パッケージ | バージョン | 分類 |
|---|---|---|
| `@astrojs/sitemap` | `3.1.4` | dependencies |
| `@biomejs/biome` | `1.8.3` | devDependencies |

その他すべての依存関係は `^x.x.x` 形式でキャレット指定。

## 問題点

- 固定した理由が不明で、将来のメンテナンス時に「なぜ固定されているのか」がわからない
- `@biomejs/biome` は頻繁にマイナーバージョンを出すツールのため、`1.8.3` に固定したままだとセキュリティ修正や新ルールが適用されない
- `@astrojs/sitemap` を固定している理由（互換性問題？バグ回避？）が追跡できない

## 対応案

### 方針 A: 固定理由をコメント等で明文化する

`package.json` にはコメントが書けないため、`README` または本 `issues/` に理由を記録する。

### 方針 B: 固定を解除してキャレット指定に戻す

互換性問題が解消されているなら `^` に戻す。

```json
// package.json
"@astrojs/sitemap": "^3.1.4",
"@biomejs/biome": "^1.8.3"
```

### 方針 C: pnpm の overrides や catalog で一元管理する

```yaml
# pnpm-workspace.yaml
catalog:
  '@astrojs/sitemap': '3.1.4'  # 理由: xxxx
  '@biomejs/biome': '1.8.3'    # 理由: xxxx
```

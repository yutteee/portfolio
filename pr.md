ライトモード/ダークモードのちらつきを修正

## 概要

テーマ切替時およびページ初期表示時に発生していたちらつきを解消するため、Header のテーマ切替実装を React state ベースから DOM/CSS ベースに変更しました。

`<html>` の `dark` クラスを単一のソースオブトゥルースとし、React 側ではテーマ状態を保持せず、クリック時に DOM と localStorage を直接操作します。これにより、SSR された HTML と React 初回レンダー間の不一致によるちらつきが発生しなくなります。

合わせて、Dev Container 内で立ち上げたサーバーをホスト PC のブラウザから開けるよう `appPort` を追加しました。

## 変更内容

- `useThemeMode` フックを削除し、`<html>` の `dark` クラスを直接トグルする実装に変更
- ダークモード用とライトモード用の `IconButton` を両方レンダーし、`:global(html.dark)` セレクターで CSS による表示切替を行うように変更
- ボタンクリック後は次に表示されるボタンへ手動でフォーカスを移すように調整（フォーカス維持の挙動を CSS 切替後も保持するため）
- 上記の実装変更に伴い Header のテスト/ストーリーを修正
  - React 側で初期テーマを判定しなくなったため、初期表示テスト（localStorage / `prefers-color-scheme` 由来の初期値）を削除
  - フォーカス遷移先が同一ボタンから別ボタンに変わるため、ストーリーのコメントを更新
- `.devcontainer/devcontainer.json` に `appPort: [4321, 6006]` を追加し、ホスト PC から Astro Dev Server / Storybook にアクセスできるように
